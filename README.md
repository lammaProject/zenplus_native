# ZenPulse

Мобильное приложение для медитации, написанное на React Native + Expo.

---

## Стек

- **React Native 0.83** + **Expo SDK 55**
- **TypeScript**
- **expo-linear-gradient** — градиенты карточек и фонов
- **react-native-safe-area-context** — корректные отступы на всех устройствах
- **Hugging Face Inference API** (Qwen2.5-72B-Instruct) — AI-генерация аффирмаций
- **Jest** + **@testing-library/react-native** — юнит-тесты

---

## Структура проекта

```
src/
  screens/
    MeditationsScreen.tsx   — главный экран с библиотекой сессий
    AIMoodScreen.tsx        — AI-настрой дня
    PaywallScreen.tsx       — экран подписки
  context/
    SubscriptionContext.tsx — глобальное состояние подписки
  data/
    sessions.ts             — данные о медитациях и категориях
  services/
    grok.ts                 — интеграция с Hugging Face API
```

## Функциональность

### Экран медитаций
- Библиотека из 10 сессий в сетке 2 колонки
- Фильтрация по категориям: Утро, Фокус, Тревога, Сон, Энергия
- Статистика: бесплатные / Premium / всего
- Баннер быстрого доступа к AI-настрою дня
- Плавный единый скролл — весь хедер часть `FlatList.ListHeaderComponent`

### AI Настрой дня
- Три настроения: Спокойный, Тревожный, Энергичный
- Генерация персональной аффирмации через Qwen2.5-72B-Instruct
- Анимация пульса во время загрузки
- Копирование аффирмации в буфер обмена

### Paywall
- Два плана: Месячный (599 ₽) и Годовой (2 990 ₽ / 249 ₽ в мес)
- Список из 6 преимуществ Premium
- Имитация покупки с модальным окном успеха

5. **Тесты** — покрытие юнит-тестами через Jest и @testing-library/react-native.

Весь цикл: идея → код → запуск → баг на эмуляторе → фикс — проходил без единой строки, написанной вручную.

## С какими специфическими проблемами мобильной верстки ИИ справляется хуже всего и как контролировалась его работа

### С чем ИИ справляется хуже всего

**1. Выравнивание разнородного контента в одной строке**

Это самая частая проблема в этом проекте. Три раза подряд один и тот же класс багов:
- Stats bar: цифра "🔒 6" рендерилась на другом уровне потому что `fontSize: 14` vs `fontSize: 18` — ИИ сам добавил `statNumLocked` и сам не подумал о последствиях для высоты строки
- Paywall: карточка без бейджа ниже карточки с бейджом — структурный дисбаланс
- AIMoodScreen: однострочное описание vs двухстрочное

ИИ пишет стили "сверху вниз" — каждый элемент сам по себе. Он плохо держит в голове как соседние элементы влияют друг на друга по высоте.

**2. Скролл с вложенными компонентами**

Классический баг — хедер зафиксирован, FlatList скроллится отдельно. ИИ написал естественную структуру `SafeAreaView → View (header) → FlatList`, не подумав что весь экран должен скроллиться как единое целое. Это интуитивно "выглядит правильно" в коде, но ломается на практике.

**3. Адаптивность под разные экраны**

В этом проекте она почти не реализована. Используется `Dimensions.get('window').width` для расчёта `CARD_WIDTH`, но этим всё и ограничивается. Под iPhone SE (320pt) карточки могут быть слишком узкими, под Pro Max — слишком большими отступами. ИИ не тестировал на разных размерах — только на одном эмуляторе Medium Phone API 36.

**4. Отладка без визуального контекста**

Первый раз когда было сказано "они на разных уровнях" — ИИ добавил `justifyContent: 'center'` и отрапортовал "готово". Только после уточнения "используй mobile-mcp" и получения точных координат элементов (y=528 vs y=535), стала видна реальная причина.

---

### Как контролировалась работа

**Что реально помогло:**

- `list_elements_on_screen` — точные пиксельные координаты, не субъективное "выглядит нормально". Именно это вскрыло что цифры стоят на y=528/535/528, а не просто "не так"
- Повторные уточнения "используй mobile-mcp" — без них ИИ останавливался на первом поверхностном фиксе
- Скриншоты после каждого изменения, не после пачки изменений

**Что не контролировалось и является риском:**

- Разные размеры экранов — всё тестировалось только на одном устройстве
- iPhone SE (375×667pt) vs Pro Max (430×932pt) — не проверялось ни разу
- Landscape orientation — не проверялось
- Системные шрифты увеличенного размера (accessibility) — не проверялось

---

### Вывод

ИИ хорошо пишет код "в вакууме" — структуры, логику, стили для одного элемента. Плохо держит системный контекст — как элементы взаимодействуют друг с другом по размеру и позиции. Контроль через инструменты инспекции (mobile-mcp coordinates) значительно эффективнее чем визуальные скриншоты, потому что цифры не врут.
